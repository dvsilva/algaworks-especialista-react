SSR (server side rendering)
    - renderização do lado do servidor
    - next.js -> permite que apenas a primeira requisição seja processada pelo servidor
SPA (single page application) aplica CSR (client side rendering)
    - renderização do lado do cliente (navegador)
    - apresenta problemas de SEO (crawlers do google / motor de busca)

SSR tradicional
    - requisição -> servidor processa html inteiro -> retorna um html inteiro
    - a cada requisição processa a página inteira
SSR com next
    - requisição -> servidor processa html inteiro -> retorna um html inteiro (com react embutido)
    - apenas a primeira requisição seja processada pelo servidor
    - não trafega documentos inteiros a cada requisição

next cli
- alternativa ao create-react-app
- permite templates e typescript
- inicializando projeto 
    - npx create-next-app alganews-blog --template typescript
    - yarn create next-app alganews-blog --template typescript
